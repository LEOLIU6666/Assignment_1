Project Idea

<

Overview

It's an AI that can analyze the recording of lecture or other speech and can write a narrative summary about which part is the important
part for the recording and which part is useless for listener and write transcript for the user. So user can find exactly the information
they want and save their time.

Motivation

I have a full-time job while doing this BP162 course and I rely on Collaborate Ultra heavily because I need to watch the recording of
the lecture I missed. But I found normally a 2-hour lecture might contain at least 10 minutes unnecessary time such as daily greeting,
computer is not working etc. And I usually did some reading before watch the recording and the lecture may contains the information
that I have already know. So I hope there is an AI can make transcript of the recording and delete al those useless staff, and I can easily
find the things I want to watch.

Describe

This application can translate the recording into words which is just like ‘real-time subtitle’, and it also can show you the whole transcript
with exact time on it. User can browse the transcript and when people see the key words which you are interested in, you can just click it
and the video will start at that certain time.

More important, this application can analyse the speech and find which part is important and which part is not related to this course and
show that on progress bar. And the lecture can be more ‘readable’ for user. For example, the professor may say “oh, I think my computer
is working now.” after fixing his computer and the application will know the professor’s pc was down and the silence time is not for
discussion which means it can be labelled as ‘not important’. Similarly, if the professor keeps repeating the same new word again and
again, the application may notice the professor is trying to introduce a new concept to us, so this part of recording should be labelled as
‘important’ or the name of this new concept.

Tools, technologies and skills

We can use Hidden Markov Model to translate the recording into context. EM algorithm is also necessary to train this Hidden Markov
Model. It can break the speech into bits which it can interpret and convert it into a digital format and analyze the piece of content. HTK
Speech Recognition Toolkit is also used for voice recognition. Additionally, Natural Language Processing (NLP) can be used to understand
the speech and try to figure out the main topic of the lecture so that it can put label on progress bar. It also can match the transcript with
PPT and improve the accuracy.

Outcome

If this application works successfully, we can watch the recording just like reading a book with catalogue. People can search the book
according to catalogue and we can find exactly the content in our recording. The user can understand the structure of this lecture and make
clear notes or summary. More importantly, user can watch the recording according to priority and save their time.
